When contributing to this repository, please first discuss the change you wish to make via issue with the maintainers of this repository before making a change. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## Code of Conduct

<Callout type="warning">
  This project and everyone participating in it is governed by the [Open Food
  Facts Code of Conduct](https://world.openfoodfacts.org/code-of-conduct),
  please follow it in all your interaction with the project. By participating,
  you are expected to uphold this code. Please report unacceptable
  behavior/abuse to abuse@openfoodfacts.org
</Callout>

## Pull Request Process

<Steps>
  <Step title="Clean Dependencies">
    Ensure any install or build dependencies are removed before the end of the
    layer when doing a build. Add only relevant files to commit and ignore the
    rest to keep the repo clean.
  </Step>
  <Step title="Update Documentation">
    Update the README.md with details of changes to the interface, this includes
    new environment variables, exposed ports, useful file locations and
    container parameters.
  </Step>
  <Step title="Request Review">
    You should request review from the maintainers once you submit the Pull
    Request.
  </Step>
</Steps>

## Git Workflow Instructions

<Callout type="info">All the commands below are to run in your console</Callout>

<Steps>
<Step title="Fork and Clone Repository">

```bash title="Repository Setup"
## Step 1: Fork Repository

## Step 2: Git Set Up & Download
# Clone the repo
git clone https://github.com/<User-Name>/facets-knowledge-panels.git

# move to the folder
cd facets-knowledge-panels

# Add upstream remote
git remote add upstream https://github.com/openfoodfacts/facets-knowledge-panels.git

# Fetch and merge with upstream/main
git fetch upstream
git merge upstream/main
```

</Step>
<Step title="Create Working Branch">

```bash title="Branch Creation"
## Create and Publish Working Branch
git checkout -b <type>/<issue-issueNumber>/{<additional-fixes>}

# Example: fix/issue-21
git push origin <type>/<issue-issueNumber>/{<additional-fixes>}
```

</Step>
<Step title="Branch Types">

<Cards>
  <Card
    title="wip"
    description="Work in Progress; long term work; mainstream changes"
  />
  <Card
    title="feat"
    description="New Feature; future planned; non-mainstream changes"
  />
  <Card title="bug" description="Bug Fixes" />
  <Card title="exp" description="Experimental; random experimental features" />
</Cards>

</Step>
<Step title="Project Setup">

<Card title="Setup Project Locally" href="/docs/Knowledge-Panel/how-to-guides/Project-setup-locally">
  Follow the project setup guide to configure your development environment.
</Card>

</Step>
<Step title="Testing">

<Card title="Testing Guide" href="/docs/Knowledge-Panel/how-to-guides/Testing">
  Test your work before pushing to ensure everything works correctly.
</Card>

</Step>
<Step title="Commit and Push">

```bash title="Commit and Push Changes"
## Committing and pushing your work

# Ensure branch
git branch

# Fetch and merge with upstream/main
git fetch upstream
git merge upstream/main

# Add untracked files
git add .

# Commit all changes with appropriate commit message and description
git commit -m "your-commit-message"

# Fetch and merge with upstream/main again
git fetch upstream
git merge upstream/main

# Push changes to your forked repository
git push origin <type>/<issue-issueNumber>/{<additional-fixes>}
```

<Callout type="warning">
**Creating Pull Request**

Create Pull Request from `<type>/<issue-issueNumber>/{<additional-fixes>}` branch in your forked repository to the main branch in the upstream repository.

After creating PR:

- Add a Reviewer (Any Admin) and yourself as the assignee
- Link Pull Request to appropriate Issue, or Project+Milestone (if no issue created)
- **IMPORTANT**: Do Not Merge the PR unless specifically asked to by an admin

</Callout>

</Step>
<Step title="After PR Merge">

```bash title="Clean Up After Merge"
# Delete branch from forked repo
git branch -d <type>/<issue-issueNumber>/{<additional-fixes>}
git push --delete origin <type>/<issue-issueNumber>/{<additional-fixes>}

# Fetch and merge with upstream/main
git checkout main
git fetch upstream
git merge upstream/main
git push origin main
```

</Step>
</Steps>

<Callout type="info">
**Ready to Contribute?**

You're now ready to start contributing to the Facets Knowledge Panels project! Remember to follow the code of conduct and feel free to ask questions in the issues if you need help.

</Callout>
