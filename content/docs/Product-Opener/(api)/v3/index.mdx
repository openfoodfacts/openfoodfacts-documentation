# Open Food Facts API v3

The next generation Open Food Facts API provides enhanced capabilities, improved performance, and modern RESTful design patterns. Built with developer experience in mind, API v3 offers cleaner endpoints, better data structures, and comprehensive validation.

<Callout type="info">

  design, better error handling, enhanced developer experience, and optimized
  performance for modern applications.
</Callout>

## Base URL

All API v3 requests use the modern endpoint structure:

```
https://world.openfoodfacts.org/api/v3
```

## Key Improvements Over v2

<Steps>
  <Step title="RESTful Design">
    Clean, predictable URL patterns following REST conventions with consistent
    HTTP methods
  </Step>
  <Step title="Enhanced Performance">
    Optimized database queries, response caching, and reduced payload sizes
  </Step>
  <Step title="Better Error Handling">
    Detailed error responses with helpful debugging information and consistent
    error codes
  </Step>
  <Step title="Modern JSON Schema">
    Improved data structures, consistent field naming, and comprehensive
    documentation
  </Step>
  <Step title="Built-in Validation">
    Request validation with clear error messages and data integrity checks
  </Step>
</Steps>

## Key Features

### Data Operations

- **Product Management**: Full CRUD operations with atomic transactions
- **Batch Processing**: Efficient handling of multiple product operations
- **Search & Filtering**: Advanced search capabilities with faceting
- **Version Control**: Track changes and maintain data history

### Integration Features

- **Authentication**: Multiple authentication methods including OAuth2
- **Rate Limiting**: Fair usage policies with clear limits and headers
- **Webhooks**: Real-time notifications for data changes
- **SDK Support**: Official client libraries for popular programming languages

<Callout type="success">
  API v3 maintains full compatibility with Open Food Facts data while providing
  a significantly improved developer experience and better performance
  characteristics.
</Callout>

## Migration from v2

<Callout type="warn">
  While migrating from API v2 to v3, note that endpoint structures and some
  response formats have changed. Review the migration guide and test thoroughly
  before switching production systems.
</Callout>

## Getting Started

1. **Explore the API**: Start with [Products](</docs/Product-Opener/(api)/v3/products>) to understand the improved data structures
2. **Authentication**: Set up authentication for write operations
3. **Test Integration**: Use the sandbox environment for development
4. **Production**: Deploy with proper error handling and rate limiting
